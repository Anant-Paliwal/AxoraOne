# Advanced Intelligence OS Skill System - Implementation Guide

## 🎯 Overview

This document describes the complete implementation of the Advanced Intelligence OS skill system - a cost-safe, event-driven, LLM-minimal skill engine.

## 📋 What Was Created

### 1. Database Migration
**File:** `ADVANCED_SKILL_SYSTEM_MIGRATION.sql`

**New Tables:**
- `skill_events` - Event-driven evidence for skill learning (NO LLM memory)
- `llm_cache` - Cache LLM outputs to minimize API calls
- `skill_cooldowns` - Enforce 24h cooldown between LLM calls per skill
- `skill_suppression` - Track suppressed skills (ignored 3+ times)

**Skills Table Upgrades:**
- `category` - planning, execution, learning, decision, research, startup
- `purpose` - User-written purpose
- `goal_type` - Array: speed, clarity, quality, focus, execution
- `scope` - page or workspace
- `evidence_sources` - JSON: pages, tasks, calendar
- `activation_signals` - Array of signals that trigger this skill
- `suggestion_types` - Array of suggestion types
- `authority_level` - read_only or suggest
- `memory_scope` - page or workspace
- `compatible_skills` - Array of compatible skill IDs
- `conflicting_skills` - Array of conflicting skill IDs
- `confidence` - Real confidence from events (0-1)
- `status` - Auto-calculated: learning, helping, reliable, trusted

**Helper Functions:**
- `calculate_skill_status(conf)` - Calculate status from confidence
- `can_call_llm(skill_id, signal, confidence)` - Check if LLM call allowed
- `update_skill_confidence(skill_id, delta)` - Update confidence and status

### 2. Skill Engine
**File:** `backend/app/services/skill_engine.py`

**Core Features:**
- Event-driven processing (NO LLM calls in main loop)
- Signal detection using simple rules
- Confidence management
- LLM call control (strict rules)
- Skill suppression (after 3 ignores)
- Confidence decay (for inactive skills)

**Signal Types:**
- `OVERSIZED_TASK` - Task delayed 2+ times with no subtasks
- `TASK_DELAYED` - Task delayed 1+ times
- `TASK_BLOCKED` - Task status is blocked
- `PAGE_NEGLECTED` - Page not updated in 30+ days
- `DEADLINE_PRESSURE` - Task due in 2 days or less

**Event Processing:**
- `process_page_event()` - Process page created/edited
- `process_task_event()` - Process task created/updated/blocked
- `process_task_completion()` - Special handler for task completion

**Confidence Rules:**
- Success: +0.05
- Ignored: -0.03
- Failed: -0.08
- Decay: -0.01 per 14 days inactivity

**Status Mapping:**
- 0-0.3 → learning
- 0.31-0.6 → helping
- 0.61-0.8 → reliable
- 0.81+ → trusted

## 🚀 How It Works

### User Creates a Skill

```
1. User fills: name, category, purpose, goal_type
2. System auto-generates:
   - activation_signals (based on category)
   - evidence_sources (pages: true, tasks: true, calendar: false)
   - authority_level (suggest)
   - memory_scope (workspace)
   - confidence (0.3)
   - status (learning)
```

### Skill Learns from Events

```
Event Occurs (page edit, task complete, etc.)
    ↓
Skill Engine detects signals (NO LLM)
    ↓
Matches signals to skills.activation_signals
    ↓
Creates skill_event record
    ↓
Updates skill.confidence
    ↓
Updates skill.status (auto-calculated)
```

### LLM Call Control (STRICT)

LLM can ONLY be called if ALL true:
1. ✅ skill.confidence >= 0.4
2. ✅ Same signal detected >= 2 times
3. ✅ Cooldown >= 24h since last call
4. ✅ Output will be shown to user

**LLM Use Cases:**
- Explaining insight
- Phrasing suggestion
- Summarizing page
- Ask Anything responses

**LLM MUST NOT:**
- Update confidence
- Trigger actions
- Activate skills

**Cache:**
- Cache key: `skill_id:signal:entity_id`
- Reuse cached outputs
- Track use count

### Skill Suppression

```
User ignores suggestion
    ↓
Increment ignore_count
    ↓
If ignore_count >= 3:
    ↓
Suppress skill for 7 days
```

### Confidence Decay

```
Every 14 days of inactivity:
    ↓
confidence -= 0.01
    ↓
status recalculated
```

from app.services.skill_engine import skill_engine
import asyncio

async def run_daily_decay():
    """Run confidence decay for all workspaces"""
    while True:
        try:
            # Get all workspaces
            workspaces = supabase_admin.table("workspaces").select("id").execute()
            
            for workspace in workspaces.data or []:
                await skill_engine.apply_confidence_decay(workspace["id"])

## 📈 Monitoring

### Key Metrics to Track

1. **Skill Confidence Distribution**
```sql
SELECT 
  status,
  COUNT(*) as count,
  AVG(confidence) as avg_confidence
FROM skills
GROUP BY status;
```

2. **Event Processing Rate**
```sql
SELECT 
  DATE(created_at) as date,
  event_type,
  COUNT(*) as events
FROM skill_events
WHERE created_at > NOW() - INTERVAL '7 days'
GROUP BY DATE(created_at), event_type
ORDER BY date DESC;
```

3. **LLM Call Frequency**
```sql
SELECT 
  DATE(last_llm_call) as date,
  COUNT(*) as llm_calls
FROM skill_cooldowns
WHERE last_llm_call > NOW() - INTERVAL '7 days'
GROUP BY DATE(last_llm_call)
ORDER BY date DESC;
```

4. **Suppressed Skills**
```sql
SELECT 
  s.name,
  ss.reason,
  ss.ignore_count,
  ss.suppressed_until
FROM skill_suppression ss
JOIN skills s ON ss.skill_id = s.id
WHERE ss.suppressed_until > NOW();
```

## 🚨 Troubleshooting

### Issue: Confidence not updating
**Check:**
1. Is task linked to skill? (`linked_skill_id`)
2. Is workspace_id set on task?
3. Check skill_events table for records
4. Check backend logs for errors

### Issue: LLM calls not working
**Check:**
1. Is confidence >= 0.4?
2. Has signal been detected 2+ times?
3. Has 24h passed since last call?
4. Check skill_cooldowns table

### Issue: Skills not detecting signals
**Check:**
1. Are activation_signals set correctly?
2. Is skill_engine imported in endpoints?
3. Are events being processed? (check logs)
4. Check signal detection rules in skill_engine.py
