# Fix: Code Blocks Not Rendering from AI Agent

## Problem

When the AI agent generates code blocks, they appear empty or don't render properly in the frontend.

## Root Cause

**Data structure mismatch:**
- Backend was generating: `{"content": "code here", "language": "python"}`
- Frontend expects: `{"code": "code here", "language": "javascript"}`

The `CodeBlockComponent` in `SimpleBlocks.tsx` reads `block.data?.code`, but the AI agents were creating blocks with `block.data?.content`.

## Files Fixed

### 1. `backend/app/services/smart_builder.py`
- Changed `"content"` → `"code"` in code block generation
- Changed default language from `"plaintext"` → `"javascript"`

### 2. `backend/app/services/agentic_agent.py`
- Fixed code block examples in AI prompts (2 locations)
- Fixed markdown-to-blocks parser to use `"code"` field

## What Changed

**Before:**
```python
"data": {
    "content": '\n'.join(code_content),
    "language": "plaintext"
}
```

**After:**
```python
"data": {
    "code": '\n'.join(code_content),
    "language": "javascript"
}
```

## Testing

1. Restart backend: `cd backend && python -m uvicorn app.main:app --reload`
2. Ask AI to create a page with code examples
3. Code blocks should now render properly with syntax highlighting

## Frontend Component

The `CodeBlockComponent` in `src/components/blocks/SimpleBlocks.tsx` correctly expects:
- `block.data?.code` - The code content
- `block.data?.language` - The programming language

Supported languages: javascript, typescript, python, html, css, json, sql, bash

## Impact

✅ Code blocks generated by AI agents now render correctly
✅ Syntax highlighting works as expected
✅ Language selector shows proper default (javascript)
✅ No frontend changes needed - backend now matches frontend expectations
